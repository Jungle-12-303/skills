# 이슈 댓글 스타일

## 기본 원칙

- 댓글은 결과 요약이어야 한다.
- "해두었습니다", "만들어두었습니다", "작업했습니다" 같은 보고체는 최대한 줄인다.
- 첫 문장에서 이슈 기준으로 반영된 결과를 바로 말한다.
- 링크는 마지막에 따로 모은다.

## 나쁜 예시

```markdown
문서 정리해두었습니다.
https://github.com/owner/repo/blob/main/docs/goal.md

주차 마무리할 때 바로 쓸 수 있게 기준을 넣어두었습니다.
```

## 좋은 예시

```markdown
이번 주 핵심 역량 목표와 달성률 산식을 `docs/goal.md`에 정리했습니다.
- 포인터/동적 할당 이해, Linked List 구현, SQL 처리기 최소 기능 구현을 기준으로 측정 항목을 다시 잡았습니다.
- 주차 종료 시 바로 계산할 수 있도록 항목별 산식을 함께 적었습니다.

관련 링크
- https://github.com/owner/repo/blob/main/docs/goal.md
```

## 문서 이슈용 템플릿

```markdown
<무엇을 정리했는지 한 줄로 요약>.
- 핵심 내용 1
- 핵심 내용 2

관련 링크
- <문서 링크>
```

## 코드 이슈용 템플릿

```markdown
<무엇을 구현/수정했는지 한 줄로 요약>.
- 동작 기준 1
- 테스트 또는 확인 내용 1

관련 링크
- <코드 링크>
- <테스트 또는 README 링크>
```
