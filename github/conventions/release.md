# 릴리즈 컨벤션

## 버전 규칙

Semantic Versioning을 따른다: `MAJOR.MINOR.PATCH`

| 버전 | 조건 |
|------|------|
| MAJOR | BREAKING CHANGE가 포함된 경우 |
| MINOR | feat 타입 커밋이 포함된 경우 |
| PATCH | fix, perf, refactor만 포함된 경우 |

## 태그 형식

```
v1.2.3
```

## 릴리즈 노트 템플릿

```markdown
## 변경 내용

### 기능 추가
- {feat 커밋 제목}
- {feat 커밋 제목}

### 버그 수정
- {fix 커밋 제목}

### 개선
- {perf, refactor 커밋 제목}

### 기타
- {chore, docs, build, ci 커밋 제목}
```

항목이 없는 섹션은 생략한다.

## 커밋 → 릴리즈 노트 변환 규칙

- 커밋 제목을 그대로 사용한다 (추가 가공 없음)
- "~했다" 어미를 "~" 명사형으로 바꾸지 않는다
- 커밋 해시는 포함하지 않는다
- PR 번호는 있으면 포함한다: `로그인 API 추가 (#42)`

## 금지 항목

style.md의 절대 금지 항목을 모두 따른다.
"이번 릴리즈에는 많은 개선이 포함되어 있습니다" 같은 서문을 추가하지 않는다.
