# README 컨벤션

## 섹션 순서

1. 프로젝트 한 줄 설명 (h1 제목 바로 아래)
2. 주요 기능 목록
3. 설치 방법
4. 사용법 (코드 예시 포함)
5. 환경 설정 (환경변수가 있는 경우)
6. 기여 방법 (오픈소스인 경우에만)

필요 없는 섹션은 생략한다. 빈 섹션을 만들지 않는다.

## 자동 업데이트 트리거

| 변경 파일 | 업데이트 섹션 |
|----------|-------------|
| scripts/*.py | 사용법 |
| .env.example | 환경 설정 |
| SKILL.md | 주요 기능 |
| conventions/*.md | 없음 (README 반영 안 함) |

## 언어 규칙

- 설명: 한국어
- 코드 예시 내부 변수명, 명령어: 영문 유지
- 섹션 제목: 한국어

## 코드 예시 규칙

실행 가능한 예시만 포함한다.
플레이스홀더는 `/path/to/...` 형태로 표기한다.
출력 예시가 있으면 주석으로 표기한다.

```bash
python3 scripts/sync_skills.py
# [완료] 동기화
#   추가됨     : commit-convention
```

## 금지 항목

style.md의 절대 금지 항목을 모두 따른다.
기존에 사람이 직접 작성한 내용을 덮어쓰지 않는다.
변경된 파일과 관련 없는 섹션은 수정하지 않는다.
