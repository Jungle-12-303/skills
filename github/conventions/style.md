# 어투 및 스타일 규칙

모든 생성 결과물(README, PR, 이슈, 릴리즈 노트)에 공통으로 적용된다.

## 절대 금지 항목

- "AI가 생성", "자동으로 작성", "Generated by", "Auto-generated" 문구
- "Co-Authored-By" 형태의 assistant 표기 footer
- 이모지 (README, PR 본문, 이슈 본문, 릴리즈 노트 전체)
- 영어 마케팅 문체 ("This PR introduces a powerful new feature...")
- 과도한 설명 ("위 내용은 다음을 의미합니다...")
- 불필요한 칭찬 ("훌륭한 변경입니다", "잘 구성된 코드입니다")
- 배지(badge), 아이콘, 장식용 이미지
- "Table of Contents" (짧은 문서에서)

## 어투 규칙

- 한국어로 작성한다. 코드, 명령어, 고유명사(라이브러리명, 프레임워크명)는 영문 유지
- 간결하게 쓴다. 한 문장으로 표현 가능하면 두 문장으로 늘리지 않는다
- 사실 중심으로 쓴다. "개선했습니다" 보다 "N ms → M ms로 줄였다"
- 종결어미는 "~한다", "~했다" 형태를 사용한다 (경어 없음)
- 수동태보다 능동태를 선호한다

## 참고 샘플 (실제 작성 예시)

아래 문장들의 문체를 참고해서 작성한다.

```
이 저장소의 스킬을 Claude와 Codex 환경에 자동으로 동기화한다.
세션 ID가 바뀌어도 자동으로 대상 경로를 탐색한다.
추가 / 업데이트 / 변경 없음 상태를 구분해서 보고한다.
인증 로직이 컨트롤러에 직접 포함되어 있어 테스트와 재사용이 어려웠다.
미들웨어 레이어로 분리해 컨트롤러는 비즈니스 로직만 담당하도록 개선한다.
```

## 길이 기준

- README 섹션 설명: 1~3문장
- PR 본문 변경 내용: 3~5문장
- 이슈 본문: 최소 정보만. 재현 방법은 번호 목록 사용
- 릴리즈 노트: 커밋 타입별로 그룹화, 각 항목 한 줄
